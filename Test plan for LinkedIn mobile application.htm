Test plan for LinkedIn mobile application

Summary:

LinkedIn application is social and professional application that concerns with professional people registration, socialization, connecting job seekers with employers, knowledge transfer, publish posts, updating person’s profile , messaging.

Testing types

We will test/focus/run following testing types: Functional testing, UI, Localization

Application modules that would be tested , and they ordered based on prioritized

1.      Registration (using Google accounts, using different email)

2.      Login

3.      Profile (update content, change account setting, privacy)

4.      Messaging

5.      Posts

6.      Notifications

7.      Networking and friends

8.      Jobs

 

High level scenarios will listed below will cover most features, but not all of them.

1)      Registration {priority 1}

a.       Validate register with existing user email /phone credentials.

b.      Check register using any email provider

c.       Check register with invalid email format.

d.      Check tap register with leaving empty all required fields.

e.      Check register using create new Google account

f.        Check register using existing Google account saved on smart Google accounts

g.       Check register using existing Google account with forget password.

h.      Validate when none of the above displayed will enable to select other registration options.

2)      Login {priority 2}

a.       Validate login with existing user with valid email and valid password

b.      Validate login with existing user with valid email and in-valid password

c.       Validate login with existing user with in-valid email and valid password.

d.      Validate tap login with empty email and empty password.

e.      Validate login with Google accounts

f.        Validate tap forget password

3)      Profile {priority 3}

a.       Check user can view his profile {priority 1}

b.      Check user can view other person profile

c.       Validate update user profile {priority 2}

d.      Validate update only one profile section or multiple sections will update this profile.

e.      Validate delete info from the profile will deleted successfully. {priority 3}

f.        Validate user can change password {priority 4}

g.       Validate user able to change his account privacy, setting, notified for what action.

4)      Post

a.       Check user can publish new post will posted/displayed on home screen

b.      Check publish new post containing image(whether from camera, or gallery ) , video , or mention a person by @sign

c.       Login with other person that mentioned on a post and check if he/she get notification of that post mentioned in.

d.      Check post comment on post will listed under post in comments section.

e.      Validate post of type “ celebrate teammate” available selections whether to give kudos , or welcome a teammate

f.        Check network and followers are able to see posts from that person if published with public visibility.

5)      Messaging

a.       Check send new message to any of connections

b.      Check send new message to person not exist in logged in user connections

c.       Check reply to existing message

d.      Validate login with other person and check if he received that new message.

e.      Validate delete message.

6)      Notifications

a.       Check navigation when tap on notification ring button.

b.      Validate tap on notification (i.e.: person commented on post) leads to that main source of notification (post comments)  

c.       Validate elements of any displayed notification are (notification source [page /person name, taken action [user has viewed your profile], action/notification time).

d.      Check available actions/settings per each notification.

e.      Validate when delete notification will be hided from notification screen.

f.        Validate un-follow posts from specific person

g.       Validate publish new post at un-followed persons/pages leads to not displayed on whether home screen or get as notification of the user who un-followed that content.

h.      Check turn off notification leads to no new notification will ring of that type again.

i.         Check when new upcoming notification ring / displayed as push notification

j.        Check tap on push notification leads to notification screen

k.       Check close app and see if still getting notifications

7)      Jobs

a.       Check tap on jobs icon at the bottom navigates to jobs screen

b.      Check navigate to jobs screen when no internet connection

c.       Validate error message displayed when no internet connection

d.      Check turn on internet connection and validate try again will refresh page content

e.      Check user can post a job  

f.        Check new posted job will displayed on his home screen

g.       Validate user able to manage his/her job search preferences

h.      Check user can edit his/her career interests

i.         Validate visibility of all his job sections whether his job preferences , new jobs based on his views and interests

j.        Validate when user select job interests will view only jobs related to that interest.

8)      General actions to be taken in each module

a.       Check add new “post , job preferences, connection , view other one profile ,login with other one credentials and check visibility , notification …. ”

b.      Check delete functionality of any feature whether post , comment , job , image, profile will delete it

c.       Check re-login again and check if deleted sections/content are really deleted or still displayed

d.      Check update on blog, post, profile or any….

e.      Check while close app will still getting notification
